HƯỚNG DẪN CẬP NHẬT HSK VIỆT V7.0 LÊN GITHUB PAGES

A. BẢO TOÀN DỮ LIỆU
1. Tạo commit hoặc sao lưu repository hiện tại.
2. Sao chép riêng js/config.js nếu đã có Supabase URL và anon key.
3. Không xóa localStorage nếu chưa xuất sao lưu slot.
4. Không xóa bảng Supabase và không chạy lại SQL.

B. CẬP NHẬT
1. Giải nén file ZIP.
2. Mở thư mục HSK-Viet-Slots-v7.0.
3. Chép toàn bộ nội dung bên trong vào repository GitHub cũ.
4. Chọn Replace khi Windows hỏi ghi đè.
5. Chép lại js/config.js đã lưu nếu file cấu hình bị thay bằng mẫu trống.
6. Commit và Push bằng GitHub Desktop.
7. Không cần npm build.
8. Sau khi GitHub Pages cập nhật, mở trang và nhấn Ctrl+F5 một lần.

C. KHÔNG CẦN NÂNG CẤP SUPABASE
- Dùng nguyên schema.sql, upgrade_v3.sql, bảng và RPC hiện tại.
- Nội dung admin cũ vẫn giữ nguyên theo word_id.
- Không đưa service_role key lên GitHub.

D. KIỂM TRA SAU KHI DEPLOY
1. Kiểm tra đủ 10 slot và tiến độ cũ.
2. Mở Thư viện, chọn một từ và kiểm tra danh sách Câu ví dụ chuẩn hóa.
3. Mở Đọc hiểu HSK 1–7–9.
4. Chuyển thử HSK 6 và HSK 7–9, chọn bài, bật/tắt pinyin và dịch từng câu.
5. Bấm một từ được highlight để xem nghĩa.
6. Nghe thử một câu và toàn bài.
7. Mở quản trị, sửa thử câu ví dụ của một từ rồi tải lại trang.
8. Nếu giao diện cũ vẫn xuất hiện, nhấn Ctrl+F5 hoặc gỡ và cài lại PWA.
